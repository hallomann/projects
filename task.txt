# Знакомство с GitHub

## Цель работы

Освоить основные команды git, научиться работать с GitHub, создать свой первий репозиторий и выполнить ряд задач для закрепления материала.

## Задание

1. **Создание учетной записи на GitHub**

    Если у вас еще нет аккаунта на GitHub, необходимо зарегистрироваться.

2. **Создание нового репозитория**

    - Зайдите на главную страницу GitHub и нажмите кнопку "New repository".
    - Назовите ваш репозиторий `hello-world`.
    - Оставьте репозиторий публичным.
    - Инициализируйте репозиторий с файлом README.

3. **Клонирование репозитория**

    Клонируйте репозиторий на свой локальный компьютер, используя Git Bash или другой терминал.


4. **Внесение изменений**

    - Перейдите в папку с клонированным репозиторием.
    - Создайте новый файл `about-me.txt`.
    - В файле напишите кратко о себе: имя, специализацию и цель изучения GitHub.
    - Сохраните файл.

5. **Добавление изменений в репозиторий**

    - Откройте терминал в папке репозитория.
    - Добавьте файл в репозиторий командой `git add about-me.txt`.
    - Сделайте коммит, используя команду `git commit -m "Add about-me file"`.
    - Загрузите изменения на GitHub с помощью команды `git push origin main`.

6. **Изучение работы с ветками**

    - Создайте новую ветку командой `git checkout -b feature-readme-instructions`.
    - Внесите изменения в файл README.md, добавив инструкции по использованию вашего проекта.
    - Сохраните файл и повторите процесс добавления изменений в репозиторий (добавьте изменения, сделайте коммит, загрузите на GitHub).

7. **Слияние веток**

    - Переключитесь обратно на ветку main командой `git checkout main`.
    - Слияйте ветку `feature-readme-instructions` с веткой `main`, используя команду `git merge feature-readme-instructions`.
    - Загрузите изменения на GitHub с помощью команды `git push origin main`.

Поздравляем! Вы выполнили ваше первое задание по работе с GitHub. Теперь вы умеете создавать репозитории, работать с ветками и сливать изменения.
